# Claude Writing｜平台风格库

日期：2026-04-10

## 用法

当用户要求使用 `claude-writing` 生成平台文案时，先判断目标平台，再按本页对应 profile 约束输出。

当前首批固定 profile：

1. `xiaohongshu_image_post`：小红书图文
2. `wechat_public_article`：微信公众号文章
3. `wechat_public_teaser`：微信公众号推送文案/标题摘要导语
4. `wechat_moments_copy`：微信朋友圈文案

## 方法说明

这套风格库不是拍脑袋总结，而是按两层来源整理：

1. 平台公开规范 / 官方可见规则
   - 用来约束不能踩的红线、卡片形态、信息流入口、封面/标题的重要程度
2. 近期运营研究与产品形态观察
   - 用来总结“这个平台用户实际上更吃什么文风”

说明：

- 关于“尺寸、流量、点击率”这类细节，如果平台没有统一公开硬规则，本页只把它作为“当前较稳的运营建议”，不是不可变平台法则。
- 关于“风格”本身，本页明确包含推断成分；凡是推断，我都尽量让它和平台分发逻辑、阅读场景、关系链强度一致，而不是凭空造风格。

## Profile 1｜小红书图文

### 基本要求

- 首屏竞争最强，用户通常先看到：
  - 标题
  - 封面
  - 前几行正文
- 内容必须更像“真实经验/真实推荐/真实踩坑”，不能像硬广通稿。
- 封面和标题不能夸大、误导、诱导点击，也不要用明显脱离内容本体的夸张表达。
- 内容里不要把读者直接往站外强导流，尤其不要把微信、私域联系方式当正文主 CTA。
- 图文更适合：
  - 清单
  - 步骤
  - 前后对比
  - 个人体验
  - 真实样例

### 文风特征

- 像一个靠谱的人在分享，而不是品牌在宣讲。
- 第一人称、具体场景、可执行建议，比抽象观点更有效。
- 句子可以短，段落最好短，允许适度口语化。
- 标题要能被“搜索”和“点击”双重理解：
  - 有具体关键词
  - 有对象/场景
  - 有结果或收益
- 常见有效框架：
  - 我怎么做
  - 我踩过什么坑
  - 哪 3 步最关键
  - 什么人适合 / 不适合

### 推荐结构

1. 标题：
   - 关键词 + 场景 + 结果
2. 首句：
   - 先给痛点或结果，不先铺背景
3. 正文：
   - 按步骤 / 清单 / 对比拆开
4. 结尾：
   - 轻互动，不要命令式强 CTA

### 不建议

- 大词空话
- 过强企业腔
- 一上来讲品牌历史
- 标题党式夸张承诺
- 太像朋友圈广告的成交口吻

## Profile 2｜微信公众号文章

### 基本要求

- 这是更适合“完整阅读”的场域，用户愿意接受更长、更完整的论证。
- 标题和封面仍然重要，但打开后决定传播的，是：
  - 结构是否清楚
  - 观点是否完整
  - 信息密度是否足够
- 文章应该能独立成立，不依赖强互动才能读完。
- 比起“强情绪勾子”，公众号更吃：
  - 清晰观点
  - 有证据的判断
  - 好的段落层次
  - 可分享的结论

### 文风特征

- 可信、完整、可转发。
- 允许更正式，但不能像公文。
- 比小红书更适合：
  - 背景
  - 分析
  - 案例
  - 结论
  - 延伸判断
- 适合“一篇文章讲透一个问题”，而不是碎片感表达。

### 推荐结构

1. 标题：
   - 明确主题，不靠悬疑吊读者
2. 导语：
   - 快速讲清为什么值得读
3. 正文：
   - 分节标题清晰
   - 每节只讲一个子问题
4. 收束：
   - 总结判断
   - 给读者一个可带走的结论

### 不建议

- 段落太碎、像社媒短帖拼接
- 标题过度情绪化
- 正文信息密度太低
- 全文只有态度，没有结构和证据

## Profile 3｜微信公众号推送文案

说明：
这里专指你说的“微信公众号”这类较短文案场景，我把它定义为：

- 推送标题
- 摘要
- 封面卡片导语
- 文章转发时的简短引导语

它和“公众号长文正文”不是一种东西。

### 基本要求

- 用户通常先在信息流/卡片里看到：
  - 标题
  - 封面
  - 摘要或很短的导语
- 所以这类文案的任务不是“讲透”，而是“让用户知道值不值得点开”。
- 必须在很短的字数里说清：
  - 这篇内容在讲什么
  - 为什么现在值得看
  - 谁该看

### 文风特征

- 短、稳、准。
- 比朋友圈更专业，比公众号正文更浓缩。
- 更像“编辑台口吻”，而不是“个人碎碎念”。
- 可以有一点情绪，但核心是信息判断，不是卖惨或煽动。

### 推荐结构

1. 标题：
   - 主题 + 结果/价值点
2. 摘要：
   - 用一句话讲清重点
3. 导语：
   - 说明读者打开后会得到什么

### 不建议

- 标题过长
- 摘要重复标题、不提供增量信息
- 摘要像空泛宣传语
- 用正文思路写标题摘要，导致信息过重

## Profile 4｜微信朋友圈文案

### 基本要求

- 朋友圈是关系链场域，不是内容分发场。
- 用户首先感受到的是“这个人今天在说什么”，不是“这条内容值不值得被算法推荐”。
- 所以朋友圈文案最重要的是：
  - 人味
  - 场景感
  - 真实感
  - 不要太像广告

### 文风特征

- 第一人称更自然。
- 适合：
  - 一个瞬间
  - 一个感受
  - 一个消息
  - 一个轻度邀请
- 比公众号更短，比小红书更私密。
- 好的朋友圈文案通常像“顺手发一条真实更新”，而不是“精心策划的公域内容”。

### 推荐结构

1. 开头：
   - 一句话带出场景或情绪
2. 中段：
   - 讲一件事或一个重点
3. 结尾：
   - 留白、轻感叹，或一个很轻的互动

### 不建议

- 大段硬卖
- 一上来就成交导向
- 太完整、太像公众号文章缩写
- 过度排比、过度鸡血、过度包装

## 快速映射

同一个主题跨平台改写时，默认这样偏：

- `小红书图文`：
  - 经验、清单、踩坑、可复制动作
- `公众号文章`：
  - 背景、分析、论证、完整结论
- `公众号推送文案`：
  - 主题提炼、摘要、点开理由
- `朋友圈文案`：
  - 人味、场景、真实更新、轻互动

## 当前结论

如果只记一句话：

- 小红书写“有用的真实分享”
- 公众号文章写“可信的完整判断”
- 公众号推送文案写“为什么值得点开”
- 朋友圈写“像本人发的真实更新”
