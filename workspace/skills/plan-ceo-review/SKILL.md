# Plan CEO Review

Use this skill when the user needs a product or founder-style judgment about whether something is worth doing, how strong the wedge is, and what tradeoffs it implies.

Before using this skill, read `../_shared/gstack_phase1_protocols.md`.

## Goal

Produce a clear product-level judgment about value, timing, prioritization, and strategic fit.

## Workflow

1. Clarify the proposal:
   - what is being considered
   - for whom
   - what outcome it is supposed to create
2. Evaluate the plan on four lenses:
   - user value
   - urgency or timing
   - leverage
   - opportunity cost
3. Identify the real bet:
   - what must be true for this to be worth doing
   - what would make it a distraction
4. Distinguish:
   - promising but not now
   - worth testing immediately
   - not worth doing
5. Recommend the next product move:
   - reject
   - narrow the scope
   - run a test
   - pass to `plan-eng-review`

## Rules

1. Judge value before implementation detail.
2. Prefer a sharp recommendation over neutral commentary.
3. Always surface what this displaces if it gets priority.
4. If the proposal is too vague, say what would need to be clarified before a product judgment is credible.
5. Do not turn a product decision into an engineering debate prematurely.

## Output Contract

Return in this order:

1. Current stage
2. What I understood
3. Product judgment
4. Why this is or is not worth doing now
5. Stage conclusion
6. Why this should move to the next stage or stop here
7. Next input needed
8. Recommended next step
